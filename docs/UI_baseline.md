# UI Baseline — 薛晓春工程作品集

> 本文档只约束视觉风格、信息层级、版式、组件外观、动效和响应式表现。
> 路由、数据模型、事实内容、构建与部署由 `engine_baseline.md` 管理。

## 1. 视觉锚点

主参考为 Zed 官网的编辑式视觉语言，但不照搬其配色和营销内容。需要继承的是：

- 贯穿页面的结构边线和统一内容框架；
- 编辑式、适中密度的文字排版；
- 大区块与真实产品画面形成的节奏；
- 少量衬线斜体作为强调，而非整页复古化；
- 克制的小按钮、小状态、细分隔线和直接的交互反馈。

当前方向为 **Zed-inspired dark editorial**：近黑背景、冰蓝强调、暖白正文、真实项目画面和 Case Study 式项目叙事。

明确避免：

- 通用 Dashboard 卡片墙；
- 大量灰色圆角卡、胶囊标签和玻璃拟态；
- 青色霓虹发光、渐变光球、代码雨、假终端和粒子背景；
- 仅靠装饰制造“科技感”；
- 精选项目与普通项目使用同一张卡片；
- 为了对称让项目卡出现宽窄不一或空洞网格。

## 2. 核心原则

1. **真实画面优先**：项目图必须来自真实架构图、运行界面、代码页面、实机照片或产品截图。
2. **证据可扫描**：角色、阶段、指标、证据等级和外链应在进入长文前被看见。
3. **首页负责选择，详情负责解释**：首页不复制详情正文，详情页也不重复营销口号。
4. **同一数据，多种呈现**：精选故事、普通项目列表和详情页可以不同版式，但事实必须一致。
5. **结构替代装饰**：用边线、列宽、留白和字号建立层级，不用堆叠阴影与发光。
6. **移动端重排而不删证据**：重点内容保留，左右结构转为上下结构。

## 3. 颜色与字体

当前语义色：

```text
页面底色       #090a0d
一级表面       #0f1116
二级表面       #171a21
默认边框       #272b34
悬停边框       #464d5c

主要文字       #f0f2f5
辅助文字       #a8adb8
弱化文字       #6f7582
交互强调       #91b5ff
低强调背景     #18284a

完成状态       #71d7a3
当前状态       #eabf74
风险状态       #ed8a8a
```

使用规则：

- 页面主体始终保持近黑，不使用大面积高饱和背景。
- 冰蓝只用于链接、焦点、关键数字和少量主动作。
- amber/green/red 只表达阶段或状态。
- 图片过亮时依靠局部深色覆盖保证文字可读，固定导航保持深色不透明度。
- 精选项目文字面板允许使用方向性半透明渐变，这是项目图片与正文的结构连接，不作为装饰光效。

字体：

```text
中文与正文：Inter, "Noto Sans SC", system-ui, sans-serif
指标与元数据："JetBrains Mono", monospace
英文强调："Instrument Serif", Georgia, serif
```

衬线体只用于英文强调句、编号和少量栏目节奏。中文标题与正文保持无衬线体。

## 4. 全局布局

- 页面主要内容框最大宽度：1240px。
- 内容框使用贯穿页面的左右细边线。
- 桌面内边距：48px；平板：32px；移动端：20px。
- Header：64px 固定高度。
- 桌面 section 垂直间距：112–128px；移动端：80px 左右。
- 普通卡片不使用大圆角；主要表面以直角和 1px 边线为主。
- 阅读正文最大宽度约 760px。

首页固定顺序：

```text
Header
个人信息与经历 Hero（作品滚动背景）
技术能力结构
精选项目
开源贡献
三版简历
联系信息 / Footer
```

## 5. Header

- Header 是页面级导航，不是首页 section 目录。
- 固定入口：品牌/首页、完整项目档案、GitHub、简历动作。
- 当前路由用文字色和底部短线共同表示。
- 品牌区域、导航和动作区使用结构分隔线。
- 固定导航背景必须足够深，不能因下方亮图导致品牌或按钮失去对比。
- 移动端使用汉堡菜单；展开层提供首页、项目档案、GitHub 和三版简历入口。
- 关键点击区不小于约 44px。

## 6. Hero

- 首屏同时出现姓名、英文视觉句、身份方向、简介、完整项目/GitHub/简历 CTA、邮箱与电话。
- 背景是来自 `projects` 数据的真实项目图片慢速横向轨道，不创建独立的装饰图片源。
- 文字侧使用深色渐变保护对比；项目画面在右侧和背景层保持可辨识。
- 桌面首屏底部显示四项已核验指标；移动端两列排列并自然延续到下一屏。
- 循环轨道在 `prefers-reduced-motion` 下停止。
- 不添加头像光环、粒子、WebGL、滚轮劫持或自定义光标。

## 7. 技术能力区

- 四个能力域使用同一横向结构表，而不是四张浮动圆角卡。
- 能力区标题下方使用单行、单向、无缝循环的官方技术徽标轨道，用于快速扫描代表性 Stack；不使用上下双轨或反向错位。
- 徽标使用统一的 40px 直角容器、克制的品牌色和一致间距；轨道悬停暂停，减少动态偏好下停止动画并允许横向浏览。
- 每个能力域显示英文域名、中文标题、编号、技能名和熟练度说明。
- 技能条目以细分隔线形成阅读节奏。
- 不使用无层级 tag 云。
- 桌面四列；平板两列；移动端单列。

## 8. 精选项目

- 数据来源为 `projects.filter(project.featured)`；数量与类别不在页面叙事中写死，可随作品集持续扩展。
- 每条为全宽项目故事，项目图片铺满整个条目背景。
- 奇数条文字在左、偶数条在右，按项目数量持续交替。
- 文字侧使用方向性深色半透明渐变；另一侧完整暴露项目图的主要视觉区域。
- 内容层级：编号/状态/证据 → 时间/角色 → 名称/副标题 → 简介 → 3 条贡献 → 1–2 个指标 → 详情入口。
- 移动端改为“图片在上、文字在下”，不强行保留左右交替。
- 图片焦点随文字方向调整，避免首图主体被渐变面板遮住。

## 9. 完整项目档案

- 独立 `/projects` 页面承载全部项目，不把筛选塞回首页。
- 页面头部动态显示当前项目与精选项目数量，并说明浏览方式，不在文案中固化数量或类别。
- 筛选为紧凑矩形按钮组：全部、系统/后端、嵌入式、机器人。
- 项目卡统一两列等宽；任何项目都不因 `featured` 改变跨度。
- 卡片结构：首图 → 状态/证据/时间 → 名称/副标题 → 截断摘要 → 指标 → 主要技术 → 角色/详情。
- 图片比例、正文区域和卡片高度保持统一；摘要过长时截断。
- 移动端单列，筛选组允许横向滚动。

## 10. 项目详情

- 顶部先显示“返回项目档案”，再显示项目状态、标题、摘要、角色和证据数量。
- 项目首图位于头部下方，使用真实资源并带说明。
- 正文采用左侧项目记录栏 + 右侧 Case Study 主栏：

```text
背景与问题
系统链路与架构取舍（如有）
方案与个人贡献
结果与证据
本机操作审计（如有）
上游与个人边界（如有）
边界与说明
技术栈
路线进展（如有）
补充视觉材料（如有）
```

- 左侧 Core stack 与正文技术栈复用同一小徽标组件；详情徽标显示图标与名称，未知技术回退为单色功能图标。

- 内容区主要依靠段落、表格行和分隔线，不把每段包成圆角卡。
- `upstream` 和 `boundaries` 必须与成果区域形成不同视觉语义。
- 路线节点同时显示符号和文字，不能只靠颜色。
- 移动端先展示主信息，再顺序堆叠项目记录栏和正文。

## 11. 开源、简历与联系

- 开源贡献使用 01–04 编号行，展示仓库、PR、摘要、贡献范围和外链箭头。
- 简历使用三列同级入口，以分隔线组织，不使用三张孤立卡片。
- 联系信息在 Hero 和 Footer 各出现一次；邮箱、电话和 GitHub 均可点击。
- Footer 使用更强的收束标题和三条直接联系方式，不添加表单。

## 12. 动效与状态

- Hero 内容首次加载使用轻量 `opacity + translateY`，总时长约 500–600ms。
- 精选项目进入视口使用约 18px 位移与淡入，只触发一次。
- 筛选切换使用 `AnimatePresence + layout`，约 200ms。
- 图片 hover 缩放不超过约 1.8%，链接箭头位移不超过 4px。
- 所有交互具有 hover、active 和 focus-visible 状态。
- 尊重 `prefers-reduced-motion`；页面功能不能依赖动效。

## 13. 响应式验收

至少检查：

```text
桌面 1440 × 900/1000
平板 768 × 1024
移动 390 × 844
```

验收要点：

- 无横向溢出、裁切、文字重叠和固定 Header 遮挡。
- Hero CTA 可换行，联系方式仍可点击。
- 技术能力区从四列自然降为两列/单列。
- 精选项目移动端图片位于正文上方，图像主体不被裁掉。
- 普通项目卡在所有筛选结果中保持统一宽度。
- 详情页标题、指标、证据、技术词和路线不会溢出。
- 汉堡菜单、筛选、路由、返回按钮、外链和下载均有明确反馈。
- 页面整体保持工程、编辑式和可信，不退化为模板化卡片墙。
